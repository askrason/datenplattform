# ADR-005: Keycloak as Central Identity Provider

## Status
Accepted

## Context
Kubernetes-native applications need unified authentication.
Keycloak is a standard OpenID Connect (OIDC) provider with user management,
group-to-role mapping, and compliance features.

## Decision
Keycloak is the single IdP for all stack components (TICKET-010).
All apps authenticate via OIDC.
User groups in Keycloak map to app roles.

## Consequences
- ✅ Single Sign-On (SSO) across all tools
- ✅ Centralized user/group management
- ✅ Fine-grained role-based access control (RBAC)
- ✅ Standards-based OIDC
- ⚠️ Keycloak downtime → SSO unavailable
- ⚠️ Client-Secret management complexity
