# ADR-004: Superset Primary, Metabase Secondary

## Status
Accepted

## Context
The team needs two BI tools:
- Superset: Full-featured, native Trino support, Keycloak OIDC
- Metabase: Simpler UX, self-service analytics, no OIDC (OSS)

## Decision
Superset is primary BI tool (TICKET-008).
Metabase is secondary, with oauth2-proxy sidecar for SSO (TICKET-009).

## Consequences
- ✅ Superset covers advanced use cases
- ✅ Metabase covers self-service
- ⚠️ Dual BI systems increase complexity
- ⚠️ Metabase users not natively in Keycloak groups (manual role sync)
