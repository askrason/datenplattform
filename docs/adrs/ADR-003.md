# ADR-003: External Secrets Operator + Vault

## Status
Accepted

## Context
Kubernetes Secrets are base64-encoded in etcd, not encrypted by default.
Production requires centralized secret management with audit logs and rotation.

## Decision
All secrets use Vault (TICKET-004) + External Secrets Operator.
No plaintext secrets in values.yaml.
All apps reference `existingSecret` fields.

## Consequences
- ✅ Centralized secret management
- ✅ Audit logs
- ✅ Dynamic secret rotation
- ✅ No K8s etcd compromise exposure
- ⚠️ Vault must be unsealed (manual or Auto Unseal)
- ⚠️ ESO adds operational complexity
