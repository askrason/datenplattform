# ADR-001: Airflow Standalone (not OpenMetadata Dependency)

## Status
Accepted

## Context
OpenMetadata 1.x deploys an embedded Airflow instance as a Chart dependency.
This instance has limited configuration and no support for custom Docker images,
making it unsuitable for general workflow management and dbt integration.

## Decision
Airflow is deployed as a standalone Sub-Chart (TICKET-005).
OpenMetadata's embedded Airflow is disabled (`airflow.enabled: false`).
OpenMetadata connects via REST API to the standalone Airflow.

## Consequences
- ✅ Full control over Airflow image (dbt + OM-Ingestion)
- ✅ KubernetesExecutor fully configurable
- ✅ Keycloak OIDC for Airflow possible
- ⚠️ Manual OM-Airflow connector configuration required
- ⚠️ OM updates may change REST API
