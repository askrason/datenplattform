# ADR-002: KubernetesExecutor for Airflow

## Status
Accepted

## Context
Airflow supports multiple executors:
- LocalExecutor: Single machine, no parallelism
- CeleryExecutor: Persistent worker pool, high resource overhead
- KubernetesExecutor: Dynamic task pods, efficient scaling

## Decision
Airflow uses KubernetesExecutor (TICKET-005).
Each task runs as an ephemeral Kubernetes Pod.

## Consequences
- ✅ No persistent worker overhead
- ✅ Better resource utilization
- ✅ Native K8s integration
- ⚠️ Scheduler needs RBAC to create/delete Pods
- ⚠️ Task startup latency (Pod creation)
